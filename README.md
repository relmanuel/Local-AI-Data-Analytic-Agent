# Local AI Data Analytic Agent 📊🤖

An offline, secure, and expert-level **AI Data Analyst and Data Scientist Agent** built using Streamlit, Pandas, Plotly, and local LLMs (via Ollama). 

This application allows you to upload any CSV dataset, ask questions in natural language, and receive precise tabular data summaries, beautiful interactive visualizations (Plotly), and structured data science insights—all running entirely locally on your machine with zero data leakage.

---

## 🌟 Key Features

* **Expert Data Scientist Prompts**: System prompts are aligned with data scientist best practices—focusing on scientific rigor, statistical significance, and business context/actionable recommendations.
* **Secure Local Execution Sandbox**: Safe execution environment that isolates and evaluates Python code generated by the LLM using a restricted environment (dangerous built-ins removed).
* **Automated Code Healing (Validator)**: Integrates a preprocessing engine that intercepts common LLM errors (e.g., missing groupby columns like `year`/`month`, generic placeholders, bad `agg` list structures, or incorrect Plotly method calls) and corrects them *before* running.
* **Intelligent Error Diagnostics**: When a KeyError occurs (such as suffix collisions `_x`/`_y` from a DataFrame merge), the sandbox automatically inspects the environment and appends descriptive hints to the LLM to auto-correct the code during retries.
* **Chronological Chart Sorting**: Auto-sorts categorical time/month axes from January to December regardless of how the data is grouped.

## 🛠️ Requirements & System Setup

To run this application, you will need to set up Python and the local Ollama LLM environment.

### 1. Prerequisite Software
* **Python**: `3.10` or higher (tested on Python `3.12`)
* **Ollama**: A local LLM server running on your system. Download and install it from [ollama.com](https://ollama.com).

### 2. Ollama Local Model Setup
Before launching the app, you need to download a code-specialized model. By default, the application is optimized for the lightweight and fast `qwen2.5-coder:1.5b` model, but you can use larger models (like `qwen2.5-coder:7b` or `llama3`) for more complex reasoning.

Start the Ollama server and run:
```bash
ollama pull qwen2.5-coder:1.5b
```

## 🚀 Installation & Getting Started

Follow these steps to set up the project locally:

### 1. Clone the Repository
```bash
git clone https://github.com/relmanuel/Local-AI-Data-Analytic-Agent.git
cd Local-AI-Data-Analytic-Agent
```

### 2. Setup Virtual Environment (Recommended)
Create and activate a virtual environment to manage dependencies:
```bash
# On MacOS/Linux:
python3 -m venv venv
source venv/bin/activate

# On Windows:
python -m venv venv
venv\Scripts\activate
```

### 3. Install Python Dependencies
Install the required packages from `requirements.txt`:
```bash
pip install -r requirements.txt
```

## 🖥️ Running the Application

Start the Streamlit dashboard by running:
```bash
streamlit run app.py
```

This will automatically launch the dashboard in your default browser at `http://localhost:8501`.

### How to Use the App:
1. **Model Name**: In the sidebar, you can see the model that is being used (default is `qwen2.5-coder:1.5b`).
2. **Upload CSV**: Upload your target dataset (e.g., e-commerce sales, user transactions, stock data) using the file uploader.
3. **Ask a Question**: Type a natural language request in the prompt area, for example:
   * *"Show me the least growing category in 2024 and tell me why"*
   * *"Plot a distribution of transaction amount by region"*
   * *"What is the trend of monthly average price in 2024?"*
4. **View Results**: The agent will run, display the **Result Data** table, generate the **Interactive Chart**, and write a detailed data scientist **Analysis & Recommendations** report at the bottom.

## 📐 Project Architecture

* **`app.py`**: The Streamlit user interface, chart configuration, and execution pipeline.
* **`agent.py`**: Handles prompts, system guidelines, code generation calls, code debugging, and the secondary analytical summaries using the local LLM.
* **`sandbox.py`**: Securely executes the generated python code, extracts the result DataFrame (`result_df`) and figure (`fig`), and parses exception tracebacks to generate diagnostic developer hints.
* **`validator.py`**: Intercepts, validates, and cleans the generated Python code, resolving common syntax mistakes, placeholder columns, or missing time/month column definitions.
* **`database.py`**: Utility to load CSV files and generate schema previews for the LLM.
